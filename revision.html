 Introduction to Python

  •  What is Python

      A high-level, interpreted, object-oriented programming language known for its simplicity and readability.

  •  Interpreter

      Software that executes Python code line by line.

  •  IDE

      Integrated Development Environment used to write and run Python programs (e.g., IDLE, VS Code).

  •  Script

      A file containing Python code, saved with .py extension.

 Python Basics

  •  Keyword

      Keywords are reserved words with special meanings in Python. Identifiers are names for variables, functions, or classes defined by the user.

  •  Identifier

      The name given to variables, functions, or classes.

  •  Variable

      Variables store data values, and data types define the type of data stored. Common types: int, float, str, list, tuple, set, and dict.

  •  Data Type

      Defines the type of value a variable can hold (int, float, str, bool).

  •  Type Casting

      Type casting converts one data type to another using functions like int(), float(), or str(). It helps ensure proper data manipulation during operations.

  •  Input Function

      A name that stores a value in memory.

  •  Output Function

      Displays output to the screen (print() function).

  •  Comment

      Notes in code ignored by the interpreter, written with #.

  •  Operator

      Operators perform operations on values or variables. Types include arithmetic, relational, logical, bitwise, assignment, membership, and identity.

 Control Statements

      Control statements direct the flow of execution. They include if, if-else, elif, loops (for, while), and jump statements (break, continue, pass).

  •  if Statement

      The if statement checks a condition — if it’s true, the code block executes. It’s used for simple decision-making.

Ex: if x > 0:print("Positive")

  •  if-else Statement

      The if-else statement executes one block if the condition is true, and another if false. It helps handle both possible outcomes.

Ex: if x % 2 == 0: print("Even") else: print("Odd")

  •  if-elif-else Statement

      Used when there are multiple conditions to test. Python checks each condition in order and executes the first one that is true.

Ex: if marks >= 90: print("A Grade") elif marks >= 75: print("B Grade") else: print("C Grade")

  •  Nested if Statement

      An if inside another if statement. Used for checking multiple dependent conditions.

Ex: if x > 0: if x < 10: print("Single Digit Positive")

  •  for Loop

      Used to iterate over a sequence like list, string, or range. It runs a block of code repeatedly for each element.

Ex: for i in range(5):print(i)

  •  while Loop

      Executes a block of code as long as a given condition is true. It’s useful when the number of iterations isn’t fixed.

Ex: while count < 5: print(count) count += 1

  •  Nested Loops

      When one loop runs inside another loop. Used for multi-level iterations, like matrices or patterns.

Ex: for i in range(3): for j in range(2): print(i, j)

  •  Break Statement

      Immediately exits the current loop, even if the condition is true. Used when a specific condition is met and no further iteration is needed.

Ex: for i in range(10): if i == 5: break

  •  continue Statement

      Skips the current iteration and moves to the next one. Used when you want to skip specific values in a loop.

Ex: for i in range(5): if i == 2: continue print(i)

  •  pass Statement

      Does nothing — acts as a placeholder to avoid syntax errors. Used when code is not yet implemented.

Ex: for i in range(3):pass

  •  Loop with else

      The else block runs when the loop finishes normally (not broken by break). Useful for checking completion conditions.

Ex: for i in range(5): print(i) else: print("Loop completed")

  •  range() Function

      Generates a sequence of numbers for looping. Syntax: range(start, stop, step)

Ex: for i in range(1, 6): print(i)

  •  Nested Control Statements

      Combination of conditional and looping statements. Used for complex logic like condition inside a loop or loop inside condition.

Ex: for i in range(3): if i % 2 == 0: print(i, "is even")

 Data Structures

      Data structures organize data efficiently. Python provides built-in structures like strings, lists, tuples, sets, and dictionaries.

  •  Strings

      Strings are sequences of characters enclosed in quotes. They are immutable and support many built-in methods for manipulation.

  •  Lists

      Lists are ordered, mutable collections of elements. They can hold mixed data types and support operations like append, remove, and sort.

  •  Tuples

      Tuples are ordered and immutable collections. They use parentheses () and are faster than lists.

  •  Sets

      Sets are unordered collections of unique elements. They support mathematical operations like union, intersection, and difference.

  •  Dictionaries

      Dictionaries store data as key-value pairs. They are mutable and fast for lookups using keys.

 Functions

      Functions are reusable blocks of code that perform a specific task. Defined using def, they improve modularity and code reusability.

  •  Function Arguments

      Arguments pass data into functions. They can be positional, keyword, default, or variable-length (*args, **kwargs).

  •  Lambda Functions

      Lambda functions are small, anonymous functions written in one line. They’re used for short, simple operations.

Ex: square = lambda x: x**2 print(square(5)) # 25

  •  Recursion

      Integrated Development Environment used to write and run Python programs (e.g., IDLE, VS Code).Recursion occurs when a function calls itself. It’s useful for repetitive problems like factorials or Fibonacci series.

  •  Higher-Order Function

      A higher-order function is a function that takes one or more functions as arguments or returns another function as its result. Examples include map(), filter(), and reduce().

Ex: def apply(func, x): return func(x)

  •  map() Function

      map() applies a given function to each item in an iterable (like a list) and returns a map object. It’s commonly used for transforming data.

Ex: nums = [1, 2, 3] result = map(lambda x: x**2, nums) print(list(result)) # [1, 4, 9]

  •  filter() Function

      filter() filters elements of an iterable based on a condition defined by a function. It returns only the elements for which the function returns True.

Ex: nums = [1, 2, 3, 4] even = filter(lambda x: x % 2 == 0, nums) print(list(even)) # [2, 4]

  •  reduce() Function

      reduce() (from functools) repeatedly applies a function to elements of an iterable to reduce it to a single value. It’s used for cumulative computations like sum or product.

Ex: from functools import reduce result = reduce(lambda x, y: x + y, [1, 2, 3, 4]) print(result)

  •  Function as Argument

      Python allows passing a function as an argument to another function. This enables dynamic execution of code blocks.

Ex: def greet(func): print("Hello", func())

  •  Function Returning Function

      A function can return another function as output. This is used in decorators and closures.

Ex: def outer(): def inner(): return "Inner Function" return inner

  •  Closures

      A closure occurs when a nested function remembers the variables from its enclosing scope even after the outer function has finished executing. Closures help retain state without using global variables.

Ex: def outer(x): def inner(): return x return inner

  •  Decorators

      Decorators modify or enhance other functions without changing their code. They’re applied using the @decorator_name syntax.

Ex: def decorator(func): def wrapper(): print("Before") func() print("After") return wrapper

 Exception Handling

      Used to handle runtime errors and prevent program crashes.

  Subtopics:

  • try — Code block where an exception might occur
  • except — Handles the exception when it occurs
  • else — Executes if no error occurs
  • finally — Executes whether error occurs or not
  • raise — Used to raise custom exceptions

Ex: try: num = int(input("Enter a number: ")) print(10 / num) except ZeroDivisionError: print("Cannot divide by zero!") except ValueError: print("Invalid input! Enter numbers only.") else: print("No error occurred.") finally: print("Program finished.")


Output: Enter a number: 0
Cannot divide by zero!
Program finished.

 Object Oriented Programming (OOPs)

      OOP is a programming method that organizes code using classes and objects.

  Subtopics:

  • Class — Blueprint for creating objects
  • Object — Instance of a class
  • Constructor — __init__() method used to initialize data
  • Encapsulation — Hiding data using private variables
  • Inheritance — Reusing properties from another class
  • Polymorphism — Same method behaves differently in different classes
  • Abstraction — Hiding complex details from the user

Ex: class Animal: def speak(self): return "Animal sound" class Dog(Animal): def speak(self): return "Bark " class Cat(Animal): def speak(self): return "Meow " d = Dog() c = Cat() print(d.speak()) print(c.speak())


Output: Bark
Meow

 Structured Query Language (SQL)

      SQL is a standard language used to store, retrieve, and manipulate data in relational databases like MySQL, Oracle, PostgreSQL, etc.

  • DDL — Data Definition Language

 1. DDL — Data Definition Language

      Used to define, create, and modify database structures like tables, schemas, or indexes.

  • CREATE — Creates database or table.
  • ALTER — Modifies table structure.
  • DROP — Deletes tables or databases.
  • TRUNCATE — Deletes all records from table but keeps structure.
  • RENAME — Renames table name.

Ex: CREATE TABLE Students (ID INT, Name VARCHAR(50), Marks INT);

 2. DML — Data Manipulation Language

      Used to manage data inside existing tables.

  • INSERT — Adds new records.
  • UPDATE — Changes existing data.
  • DELETE — Removes data.

Ex: SELECT * FROM Students WHERE Marks > 80;

 3. DCL — Data Control Language

      Used to control access permissions in a database.

  • GRANT — Provides user privileges.
  • REVOKE — Removes privileges.

Ex: GRANT SELECT ON Students TO user1;

 4. TCL — Transaction Control Language

      Used to manage transactions in a database.

  • COMMIT — Saves all changes.
  • ROLLBACK — Undoes last changes.
  • SAVEPOINT — Marks a point in a transaction.
  • SET TRANSACTION — Defines transaction properties.

Ex: COMMIT;

 5. DQL — Data Query Language

      Used to extract data from existing tables.

  • SELECT — Retrieves data from tables.

Ex: SELECT * FROM Students WHERE Marks > 80;

 6. Constraints

      Used to set rules for data in a table.

  • PRIMARY KEY — Uniquely identifies each record.
  • FOREIGN KEY — Links two tables.
  • UNIQUE — Ensures all values are unique.
  • NOT NULL — Field cannot be empty.
  • CHECK — Restricts values in a column.
  • DEFAULT — Sets default value if none provided.

Ex: CREATE TABLE Emp(ID INT PRIMARY KEY, Salary INT CHECK(Salary>5000));

 7. Joins

      Used to combine rows from two or more tables.

  • INNER JOIN — Returns matching records.
  • LEFT JOIN — Returns all from left table.
  • RIGHT JOIN — Returns all from right table.
  • FULL JOIN — Returns all records from both tables.
  • SELF JOIN — Joins table to itself.
  • CROSS JOIN — Returns all combinations.

Ex: SELECT e.Name, d.DeptName FROM Emp e INNER JOIN Dept d ON e.DeptID=d.DeptID;

 8. Clauses

  • WHERE — Filters records.
  • ORDER BY — Sorts records.
  • GROUP BY — Groups rows with same values.
  • HAVING — Filters groups after grouping.
  • LIMIT — Restricts number of rows.

Ex: SELECT DeptID, COUNT(*) FROM Emp GROUP BY DeptID HAVING COUNT(*)>5;

 9. Functions

      Built-in SQL functions to perform calculations.

  • COUNT(), SUM(), AVG(), MIN(), MAX()
  • String: UPPER(), LOWER(), SUBSTRING()
  • Date: NOW(), CURDATE()

 10. Subqueries

      Query inside another query.

Ex: SELECT Name FROM Emp WHERE Salary > (SELECT AVG(Salary) FROM Emp);

 11. Views

      A virtual table created using a query.

Ex: CREATE VIEW HighSalary AS SELECT * FROM Emp WHERE Salary > 50000;

 12. Indexes

      Used to speed up retrieval of records.

Ex: CREATE INDEX idx_name ON Emp(Name);
